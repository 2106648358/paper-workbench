# Design: Three-Line Table Optimization

## Context

### Background
论文《HIIT对舞蹈啦啦操运动员专项技术动作提升的实验研究》包含16个表格，需要按照三线表规范进行格式优化。规范文档已创建于 `docs/三线表规范.md`。

### Current State
- 论文已使用 `booktabs` 宏包
- 部分表格使用了多余的 `\midrule` 进行分组
- 线条粗细使用默认值（未设置）
- 表格编号和标签已正确设置

### Constraints
- 不能改变表格数据内容
- 必须保持表格编号和标签不变
- 需要保持行文逻辑的连贯性

---

## Goals / Non-Goals

**Goals:**
- 所有表格符合三线表规范
- 线条粗细符合标准（顶线/底线 1.5pt，栏目线 0.75pt）
- 过宽/过长表格得到妥善处理
- 行文逻辑保持连贯

**Non-Goals:**
- 不修改表格数据
- 不改变表格编号和标签
- 不重新设计表格结构

---

## Decisions

### Decision 1: 导言区配置

**Chosen**: 在导言区添加三线表线条粗细配置

**Implementation**:
```latex
% 在 \usepackage{booktabs} 之后添加
% ==================== 三线表配置 ====================
\setlength{\heavyrulewidth}{1.5pt}   % 顶线和底线
\setlength{\lightrulewidth}{0.75pt}  % 栏目线
\setlength{\cmidrulewidth}{0.4pt}    % \cmidrule

% 自定义命令（可选）
\newcommand{\doubleline}{\midrule[0.75pt]}  % 双竖细线
```

**Position**: 在第18行 `\usepackage{booktabs}` 之后添加

---

### Decision 2: 多余 midrule 处理策略

**Chosen**: 逐表分析，分类处理

| 情况 | 处理方式 |
|------|----------|
| 数据分组明确（如按技术类型分组） | 移除 midrule，在行文中补充说明 |
| 数据无明确分组 | 直接移除多余 midrule |
| 需要保留分组信息 | 使用 `\cmidrule` 替代或添加文字说明 |

---

## Risks / Trade-offs

### Risk 1: 分组信息丢失
- **Risk**: 移除分组横线后，读者可能难以区分不同组别的数据
- **Mitigation**: 在表格前后的行文中补充说明，或使用缩进/空格区分

### Risk 2: 版面变化
- **Risk**: 移除横线后表格高度减小，可能影响版面布局
- **Mitigation**: 编译后检查版面，必要时调整文字内容

### Risk 3: 表格引用错误
- **Risk**: 修改表格后，正文中对表格行列的引用可能失效
- **Mitigation**: 逐段检查引用表格的段落文字

## Open Questions

1. **是否需要保留任何分组横线？**
   - 如果某些表格的分组信息非常重要，可以考虑使用 `\cmidrule` 替代
   - **Decision**: 先尝试完全移除，如果影响理解再添加文字说明

2. **附录中的表格是否需要调整？**
   - 附录中的评分标准表格可能是参考性质，格式要求可能不同
   - **Decision**: 统一按三线表规范处理
